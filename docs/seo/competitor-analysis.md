# SEO Competitor Analysis

## Overview
This document contains the ChatGPT prompts and strategy for competitor analysis for blindfoldcubing.com.

## Application Context

### Our Website: blindfoldcubing.com
- **Primary Function**: Web-based training application for blindfold cubers
- **Key Features**: 
  - Multiple training drills (Flash Pairs, Edge/Corner Notation, Edge Tracing, Memorization)
  - Progress tracking and analytics
  - Session management with recall accuracy measurement
  - Offline-first, mobile-responsive web app
  - Free to use, no registration required (optional Firebase auth)
- **Current Content**:
  - Main training application (/)
  - Beginner's guide to blindfold cubing (/intro.html)
  - Interactive onboarding tutorial (/onboarding.html)
- **Target Audience**: Beginners to advanced blindfold cubers
- **Unique Selling Points**: 
  - Structured progressive training drills
  - Multiple drill types for different skills
  - Offline capability
  - Progress tracking and analytics
  - Free and accessible

### Our Content Strengths
- Comprehensive beginner's guide
- Interactive training tools
- Multiple drill types
- Progress tracking features
- Offline-first design

### Our Content Gaps (Potential)
- Advanced techniques and strategies
- Detailed Speffz notation tutorials
- Memory palace technique deep-dives
- Edge/corner tracing detailed guides
- Comparison with other training methods
- FAQ sections
- Video tutorials or visual guides
- Community resources

## Phase 1: Content Gap Analysis

### ChatGPT Prompt 2.1
```
I'm conducting a content gap analysis for blindfoldcubing.com, a training application for blindfold cubers.

Our Website (blindfoldcubing.com):
- Free web-based BLD training tool
- Features: Flash Pairs, Edge/Corner Notation drills, Edge Tracing, Memorization drills
- Current Content: Main training app, beginner's guide (/intro.html), onboarding tutorial
- Target Audience: Beginners to advanced blindfold cubers
- Unique Features: Multiple drill types, progress tracking, offline-first, session management

Our Current Content Covers:
- Basic BLD introduction and process
- Memory system and letter pairs overview
- Learning progression (5 steps)
- Memory Palace technique introduction
- Speffz notation basics
- Application features and training modes

Conduct a content gap analysis by analyzing our competitor: [Competitor website URL]

Identify topics and subtopics they've covered extensively that we have NOT covered, focusing on:
1. Educational content about blindfold cubing (advanced techniques, strategies, tips)
2. Training techniques and methods (different approaches, best practices)
3. Memory techniques for BLD (detailed memory palace, visualization methods, story building)
4. Speffz notation tutorials (detailed guides, edge/corner notation, tracing methods)
5. Edge and corner tracing guides (step-by-step tutorials, common mistakes)
6. FAQ content (common questions, troubleshooting, beginner mistakes)
7. Video or visual content (tutorials, demonstrations, walkthroughs)
8. Community resources (forums, discussions, user experiences)
9. Comparison content (training methods, tools, techniques)
10. Advanced topics (speed optimization, accuracy improvement, competition strategies)

For each gap identified, provide:
- Topic/Subtopic name
- Content type (Guide/Tutorial/FAQ/Video/Comparison)
- Why it's valuable (user need it addresses)
- Estimated search volume potential (High/Medium/Low)
- Difficulty to create (Easy/Medium/Hard)
- How it relates to our training application
```

## Phase 2: Competitor Keyword Identification

### ChatGPT Prompt 2.2
```
I'm analyzing competitor keywords for blindfoldcubing.com to find ranking opportunities.

Our Website:
- Domain: blindfoldcubing.com
- Application: Free BLD training tool with multiple drill types
- Features: Flash Pairs, Edge/Corner Notation, Edge Tracing, Memorization drills, progress tracking
- Current Rankings: New website, limited SEO history
- Strengths: Comprehensive training tools, offline-first, progress tracking, free access

Competitor to Analyze: [Competitor website URL]

Generate a list of keywords where [competitor] currently outranks blindfoldcubing.com but has vulnerabilities we can exploit, such as:

1. **Content Quality Issues**:
   - Low-quality or outdated content
   - Thin content (lacks depth)
   - Missing important information
   - Poor explanations or unclear instructions

2. **Technical SEO Issues**:
   - Missing structured data (JSON-LD)
   - Poor mobile responsiveness
   - Slow page load times
   - Missing meta tags or poor meta descriptions

3. **User Experience Issues**:
   - Poor navigation or site structure
   - Difficult to use interface
   - Missing features users want
   - No progress tracking or analytics

4. **Content Gaps**:
   - Missing visual content (images, videos)
   - No interactive elements
   - Lack of practical examples
   - Missing step-by-step guides

5. **Link Building Issues**:
   - Weak internal linking
   - No external backlinks
   - Missing related content links
   - Poor anchor text usage

For each vulnerable keyword, provide:
- Keyword phrase
- Current competitor ranking (estimated)
- Vulnerability type (Content Quality/Technical SEO/UX/Content Gap/Linking)
- Why we can outperform (our advantage)
- Our relevant features/tools
- Content strategy to rank for this keyword
- Estimated difficulty to rank (Easy/Medium/Hard)
```

### Follow-up Prompt 2.2
```
From the list of vulnerable keywords you identified, create a prioritized list with low-hanging fruit keywords at the top.

Prioritization Criteria:
1. **Ease of Ranking** (Easy/Medium/Hard) - keywords we can rank for quickly
2. **Search Volume** (High/Medium/Low) - how many people search for this
3. **User Intent Match** (High/Medium/Low) - how well it matches our training app
4. **Competitor Vulnerability** (High/Medium/Low) - how weak the competitor's content is
5. **Our Competitive Advantage** (High/Medium/Low) - how well we can outperform

For the top 10 priority keywords, provide:

1. **Content Strategy**: What type of content should we create? (Guide/Tutorial/FAQ/Tool Page/Blog Post)
2. **Content Outline**: Key sections and topics to cover (3-5 main sections)
3. **Our Features to Highlight**: Which training app features are most relevant
4. **Technical SEO**: What structured data, meta tags, or technical optimizations needed
5. **Internal Linking**: Which existing pages should link to this content
6. **Competitive Advantage**: Why our content would be better than the competitor's
7. **Timeline Estimate**: How long to create and optimize this content
8. **Expected Impact**: Estimated traffic and ranking potential

Also, suggest 3-5 long-tail keyword variations for each top priority keyword that we could target simultaneously.
```
