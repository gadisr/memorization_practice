6. Recommended Homepage Layout (Priority Order)

  1. Hero Section - Main value prop + "Start Free Drill" button
  2. Quick Drill Interface - Let users try immediately (no sign-in required)
  3. Progress Teaser CTA - Show their activity, prompt sign-in to save
  4. Drill Modules Grid - Flash Pairs, Memory Palace, etc.
  5. PDF Tutorial Section - Sign-in gate for download
  6. Social Proof Section - Stats, testimonials, or user count
  7. Features Overview - What they get when signed in
Key Principle: Let users experience value first (free drill), then show them what they're missing (progress tracking), then make it easy to sign in.

7. Implementation Notes

Best Practices:

  • Show progress teaser after user completes 1-2 drills (creates investment)
  • Use the loss aversion CTA immediately after first drill completion
  • Place PDF download section prominently but require sign-in
  • Make sign-in buttons large, clear, and use Google OAuth (low friction)
  • Show benefits clearly: "Save progress", "Track stats", "Download PDF"
  • Use social proof numbers if available (e.g., "2,847 cubers tracking progress")
  • Don't gate the initial drill experience - let them try first
